Create `app/api/association_routes.py` with a FastAPI router (prefix="/associations", tags=["associations"]).

Follow the same patterns as `catalog_routes.py` and `segments_routes.py`: import `get_supabase` from `app.db`, import `ProfileContext, get_profile_context` from `app.api.auth`, use `Depends(get_profile_context)` on all routes.

**Pydantic models:**

```python
class AssociationCreate(BaseModel):
    segment_id: str
    catalog_product_id: str
    selected_image_urls: list[str] = []

class AssociationUpdate(BaseModel):
    selected_image_urls: list[str]

class AssociationResponse(BaseModel):
    id: str
    segment_id: str
    catalog_product_id: str
    selected_image_urls: list[str]
    pip_config: Optional[dict] = None
    slide_config: Optional[dict] = None
    created_at: str
    updated_at: str
    # Joined product info
    product_title: Optional[str] = None
    product_image: Optional[str] = None
    product_brand: Optional[str] = None
```

**Endpoints:**

1. **POST /associations** (ASSOC-01: associate product with segment)
   - Accepts `AssociationCreate` body
   - Validates segment exists in `editai_segments` (query by id, filter by profile_id from context)
   - Validates product exists in `v_catalog_products_grouped` (query by id)
   - Upserts into `segment_product_associations` (ON CONFLICT on segment_id → update product + images)
   - Since Supabase client supports `upsert()`, use: `supabase.table("segment_product_associations").upsert(data, on_conflict="segment_id").execute()`
   - Returns the created/updated association with product details enriched

2. **GET /associations/segment/{segment_id}** (ASSOC-03: view association)
   - Fetches association from `segment_product_associations` by `segment_id`
   - If found, enriches with product info from `v_catalog_products_grouped` (title, image_link, brand)
   - Returns `AssociationResponse` with product details, or 404 if no association

3. **GET /associations/segments** (batch version of ASSOC-03)
   - Query param: `segment_ids` (comma-separated UUIDs)
   - Fetches all associations for the given segment IDs in one query
   - Enriches each with product info
   - Returns `{"associations": {segment_id: AssociationResponse | null, ...}}`
   - This batch endpoint prevents N+1 queries when loading the segments page

4. **PATCH /associations/{association_id}** (ASSOC-04: update selected images)
   - Accepts `AssociationUpdate` body
   - Updates `selected_image_urls` on the association
   - Returns updated association

5. **DELETE /associations/segment/{segment_id}** (ASSOC-02: remove association)
   - Deletes the association for a segment
   - Returns `{"deleted": true, "segment_id": segment_id}`
   - 404 if no association exists for that segment

**Error handling**: Use try/except around Supabase calls, log errors, return appropriate HTTP status codes (400 for validation, 404 for not found, 500 for DB errors).

**Profile scoping**: For POST and GET endpoints, verify the segment belongs to the current profile by checking `editai_segments.profile_id` matches `profile.profile_id` from context. This prevents cross-profile data access.
