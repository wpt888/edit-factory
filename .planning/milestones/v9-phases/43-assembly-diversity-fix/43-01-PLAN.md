<objective>
Fix the assembly pipeline so video segments never repeat until all available segments have been used, and segments from the same source video with overlapping time ranges do not appear consecutively.

Purpose: The current merge step in `build_timeline` picks ONE representative per merged group, destroying the segment diversity that `match_srt_to_segments` carefully established via round-robin. This causes the same clip to repeat visually even when the pool has unused segments.

Output: A corrected `assembly_service.py` where the merge step produces sub-entries (one per original SRT match within the group) so each segment gets its proportional screen time, and an adjacency check prevents same-source overlapping-time segments from appearing back-to-back.
</objective>

  <action>
Rewrite the merge step in `build_timeline` (the `if min_segment_duration > 0 and len(timeline) > 1:` block, approximately lines 796-852) to preserve all segment assignments rather than collapsing to one representative.

**Current behaviour (broken):** Merges N short entries into 1 entry using one representative segment. Diversity is destroyed.

**New behaviour:** When merging N short entries into a group:
1. Keep all N original `TimelineEntry` objects as "sub-entries" within the merged group
2. Redistribute the `accumulated_duration` proportionally across the sub-entries (each sub-entry gets `accumulated_duration * (its_original_duration / sum_of_original_durations)`)
3. Each sub-entry keeps its own `source_video_path`, `start_time`, `transforms`
4. Update each sub-entry's `timeline_start` to be sequential within the group
5. Flatten the result: the final `timeline` list is a flat list of TimelineEntry objects where merged groups simply appear as consecutive entries with corrected durations and timeline positions

**Implementation approach:**
Replace the current merge block with:

```python
if min_segment_duration > 0 and len(timeline) > 1:
    merged = []
    i = 0
    while i < len(timeline):
        current = timeline[i]
        accumulated_duration = current.timeline_duration
        last_merged_idx = i

        # Absorb following entries while under minimum
        while accumulated_duration < min_segment_duration and last_merged_idx + 1 < len(timeline):
            last_merged_idx += 1
            accumulated_duration += timeline[last_merged_idx].timeline_duration

        # Collect all sub-entries in this merged group
        sub_entries = timeline[i:last_merged_idx + 1]
        original_total = sum(e.timeline_duration for e in sub_entries)

        # Redistribute accumulated_duration proportionally across sub-entries
        group_timeline_start = current.timeline_start
        for se in sub_entries:
            proportion = se.timeline_duration / original_total if original_total > 0 else 1.0 / len(sub_entries)
            new_duration = accumulated_duration * proportion
            merged.append(TimelineEntry(
                source_video_path=se.source_video_path,
                start_time=se.start_time,
                end_time=se.start_time + new_duration,
                timeline_start=group_timeline_start,
                timeline_duration=new_duration,
                transforms=se.transforms,
            ))
            group_timeline_start += new_duration

        i = last_merged_idx + 1

    # If the last entry is shorter than minimum, absorb it into
    # the previous entry (keep existing logic for this edge case)
    if len(merged) >= 2 and merged[-1].timeline_duration < min_segment_duration:
        last = merged.pop()
        prev = merged[-1]
        combined_duration = prev.timeline_duration + last.timeline_duration
        merged[-1] = TimelineEntry(
            source_video_path=prev.source_video_path,
            start_time=prev.start_time,
            end_time=prev.start_time + combined_duration,
            timeline_start=prev.timeline_start,
            timeline_duration=combined_duration,
            transforms=prev.transforms,
        )

    logger.info(
        f"Merged timeline: {len(timeline)} entries -> {len(merged)} entries "
        f"(min_segment_duration={min_segment_duration}s)"
    )
    timeline = merged
```

**Key difference:** Instead of picking ONE representative, ALL sub-entries survive with proportionally redistributed durations. The flat list preserves every segment's screen time.

Also update the "last merged group is too short" tail-absorption to still work correctly — it absorbs the final entry into the previous one (this is acceptable because it only triggers for the very last entry).

Enhance the adjacency prevention in `match_srt_to_segments` to check not just `source_video_id` equality but also whether two consecutive segments from the same source have overlapping time ranges.

**Current behaviour:** The round-robin `_rr_next` and keyword matching use `exclude_source_video_id` to prefer a different source video entirely. But two segments from the same source video are fine as long as their time ranges don't overlap (e.g., segment at 0-5s and segment at 30-35s from the same source are visually distinct).

**New behaviour:** Track `prev_segment_start_time` and `prev_segment_end_time` alongside `prev_source_video_id`. When evaluating candidates:
- If candidate has the same `source_video_id` as previous AND their time ranges overlap (candidate.start < prev.end AND prev.start < candidate.end), treat as "same visual content" and try to avoid.
- If candidate has different `source_video_id`, always acceptable.
- If candidate has same `source_video_id` but non-overlapping time range, acceptable.

**Implementation changes in `match_srt_to_segments`:**

1. Add tracking variables after `prev_source_video_id`:
```python
prev_segment_start: Optional[float] = None
prev_segment_end: Optional[float] = None
```

2. Create a helper function:
```python
def _overlaps_previous(seg: dict) -> bool:
    """Check if segment overlaps with the previous segment's time range on the same source."""
    if not prev_source_video_id or seg.get("source_video_id") != prev_source_video_id:
        return False
    if prev_segment_start is None or prev_segment_end is None:
        return False
    seg_start = seg.get("start_time", 0.0)
    seg_end = seg.get("end_time", 0.0)
    return seg_start < prev_segment_end and prev_segment_start < seg_end
```
