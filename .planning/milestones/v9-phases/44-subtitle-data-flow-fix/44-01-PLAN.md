<objective>
Persist SRT subtitle content from Step 2 (preview) into the tts_previews cache so Step 3 (render) can reuse it without regenerating TTS audio.

Purpose: Currently tts_previews stores audio_path, audio_duration, script_hash, voice_settings, and words_per_subtitle — but NOT srt_content. This causes the render step to call ElevenLabs a second time just to get timestamps for SRT regeneration, wasting API credits and introducing potential timing drift between preview and render subtitles.

Output: Modified pipeline_routes.py with srt_content persisted into tts_previews at two points: (1) after preview_variant completes, and (2) after standalone TTS generation. The render path already reads `existing_tts.get("srt_content")` at line 1334 — it just needs the data to actually be there.
</objective>

  pipeline["tts_previews"][variant_index]["srt_content"] = preview_data.get("srt_content", "")
  # Also persist audio info from preview_data if tts_previews was empty
  if not pipeline["tts_previews"][variant_index].get("audio_path"):

      pipeline["tts_previews"][variant_index]["timestamp"] = datetime.now(timezone.utc).isoformat()
