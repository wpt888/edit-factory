Add a new exported interface and update the timeline editor:

1. **Define InterstitialSlide type** (export it for use in pipeline page):
```typescript
export interface InterstitialSlide {
  id: string;                    // Unique ID (use crypto.randomUUID())
  afterMatchIndex: number;       // Insert after this match index (-1 = before first)
  imageUrl: string;              // Product image URL
  duration: number;              // Seconds (default 2.0, range 0.5-5.0)
  animation: "static" | "kenburns"; // Ken Burns or static (default "kenburns")
  kenBurnsDirection?: "zoom-in" | "zoom-out" | "pan-left" | "pan-right"; // Default "zoom-in"
  productTitle?: string;         // For display in timeline
}
```

2. **Extend TimelineEditorProps** with two new optional props:
```typescript
interstitialSlides?: InterstitialSlide[];
onInterstitialSlidesChange?: (slides: InterstitialSlide[]) => void;
```

3. **In the timeline view (horizontal strip)**, between each group block and also before the first / after the last, render a small "+" button:
   - The "+" button should be a thin vertical line/button (16px wide, full height of the strip) with a `Plus` icon
   - Styled subtly: dashed border, muted color, hover highlight (purple/indigo to distinguish from segment colors)
   - On click: call a `handleInsertSlide(afterMatchIndex)` function that creates a new InterstitialSlide with default values and appends to the slides array via `onInterstitialSlidesChange`
   - The `afterMatchIndex` is the last match index of the preceding group (-1 if inserting before everything)

4. **Render interstitial slide blocks** in the timeline strip:
   - After the "+" button, if there's an InterstitialSlide at that position, render it as a block similar to segment blocks but with distinct styling (purple/indigo background, dashed border)
   - Show: small product image thumbnail (if imageUrl), duration label, "KB" badge if kenburns animation
   - Clicking the slide block selects it (use a new `selectedSlideId` state, mutually exclusive with `selectedBlockIndex`)
   - Width proportional to duration relative to totalDuration (same as segment blocks)

5. **Inline slide config panel** (shown when a slide block is selected, similar to the segment detail panel):
   - Product image preview (small thumbnail)
   - Image URL input field (text input, pre-filled)
   - Duration slider: 0.5s to 5.0s, step 0.5, with +/- buttons (reuse the existing adjustDuration pattern)
   - Animation toggle: "Static" / "Ken Burns" buttons (like PipOverlayPanel animation options)
   - If Ken Burns selected: direction dropdown (zoom-in, zoom-out, pan-left, pan-right)
   - "Remove" button (red, removes the slide from array)
   - All changes call `onInterstitialSlidesChange` with updated slides array

6. **In the list view**, render interstitial slides as special rows between the appropriate match rows:
   - Purple left border, distinct background
   - Show image thumbnail, duration, animation type, remove button
   - "+" buttons between list rows for insertion (same logic as timeline view)

Important implementation notes:
- Do NOT import `crypto` — use `Math.random().toString(36).slice(2, 10)` for generating slide IDs (works in all browsers)
- The "+" buttons should only appear when `onInterstitialSlidesChange` is provided (feature is opt-in)
- Interstitial slides do NOT participate in drag-and-drop (they have fixed positions relative to matches)
- Import `ImageIcon` from lucide-react for the slide blocks (avoid name collision with HTML Image)
