**Layout — vertical list of timeline rows:**
Each row represents one SRT phrase and its segment assignment. Display:
- Left column: SRT phrase index (1-based), time range formatted as "0:02 - 0:05"
- Center column: SRT text (truncated to ~60 chars with title for full text)
- Right column: Matched segment info — show matched_keyword badge (green) and confidence %, OR "Unmatched" badge (amber/warning) with a "Select Segment" button

**Visual styling:**
- Matched rows: left border green (`border-l-4 border-green-500`), light green background (`bg-green-50 dark:bg-green-950/20`)
- Unmatched rows: left border amber (`border-l-4 border-amber-500`), light amber background (`bg-amber-50 dark:bg-amber-950/20`)
- Rows stacked vertically with 1px gap between them
- Each row is ~48px tall, scrollable container with max-height

**Manual segment assignment dialog:**
When user clicks "Select Segment" on an unmatched phrase:
1. Open a Dialog (use existing shadcn Dialog component)
2. Fetch segments from `GET /api/v1/segments/list?profile_id={profile}&source_video_ids={ids}` — or if that endpoint doesn't exist, fetch from the existing segments data passed as prop
3. Actually, since the pipeline page already has access to segment data via the preview API, pass available segments as a prop instead: `availableSegments: {id: string, keywords: string[], source_video_id: string}[]`
4. Show a searchable list of segments with their keywords
5. On select, update the match: set segment_id, segment_keywords, matched_keyword (first keyword), confidence to 1.0
6. Call `onMatchesChange` with the updated matches array
