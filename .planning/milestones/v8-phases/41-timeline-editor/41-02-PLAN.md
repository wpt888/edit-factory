const handleDrop = (dropIndex: number) => {
  if (dragIndex === null || dragIndex === dropIndex) return;
  const updated = [...matches];
  const [moved] = updated.splice(dragIndex, 1);
  updated.splice(dropIndex, 0, moved);
  // Re-index srt_index values to match new order

6. **Important:** When reordering, the SRT timing (srt_start, srt_end) stays with the original phrase text — only the segment assignment (segment_id, segment_keywords, matched_keyword, confidence) swaps position. Actually, the entire MatchPreview object moves (text + segment mapping together), since we're reordering the visual sequence. The srt_index gets re-numbered to reflect new order.

Actually, reconsidering: The user wants to reorder SEGMENTS (which video plays when), not reorder the SRT phrases. The SRT phrases have fixed timing from the audio. So what should happen is: the segment assignments move while the phrase rows stay fixed.

**Corrected approach — swap segment assignments:**
When dragging row A to row B's position:
- Row A keeps its SRT text/timing but gets Row B's segment assignment
- Row B keeps its SRT text/timing but gets Row A's segment assignment
- This effectively says "play segment X during phrase Y instead of phrase Z"

```typescript
const handleDrop = (dropIndex: number) => {
  if (dragIndex === null || dragIndex === dropIndex) return;
  const updated = [...matches];
  // Swap segment assignments between dragged and dropped positions
  const dragSegment = {
    segment_id: updated[dragIndex].segment_id,
    segment_keywords: updated[dragIndex].segment_keywords,
    matched_keyword: updated[dragIndex].matched_keyword,
    confidence: updated[dragIndex].confidence,
  };
  const dropSegment = {
    segment_id: updated[dropIndex].segment_id,
    segment_keywords: updated[dropIndex].segment_keywords,
    matched_keyword: updated[dropIndex].matched_keyword,
    confidence: updated[dropIndex].confidence,
  };
  updated[dragIndex] = { ...updated[dragIndex], ...dropSegment };
  updated[dropIndex] = { ...updated[dropIndex], ...dragSegment };
  onMatchesChange(updated);
  setDragIndex(null);
  setDragOverIndex(null);
};
```

This swap-based approach is more intuitive: "drag this segment to play during that phrase instead."
