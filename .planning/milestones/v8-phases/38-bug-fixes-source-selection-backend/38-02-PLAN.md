<objective>
Add source-scoped segment matching to the pipeline API so preview and render only use segments from user-selected source videos.

Purpose: Currently the pipeline matches SRT phrases against ALL segments in the profile's library. When a user has many source videos, irrelevant segments dilute match quality. This adds a `source_video_ids` parameter so the frontend (Phase 39) can pass the user's source video selection and get focused, higher-quality matches.

Output: Updated preview and render endpoints that accept an optional `source_video_ids` list, and assembly service methods that filter segments accordingly.
</objective>
