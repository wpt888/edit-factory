## Goal

Build the `/setup` page as a 3-step wizard (license activation, API key configuration, crash reporting consent), hide the navbar on `/setup`, and add a "Setup Wizard" button to the Settings page for re-entry. On first launch (desktop mode), users are guided through configuration before accessing the app.

## Context

- Phase 49 built all backend endpoints: `POST /license/activate`, `POST /license/validate`, `GET /settings`, `POST /settings`
- Plan 50-01 adds: `POST /first-run/complete`, `POST /test-connection`, and extends `GET /settings` with `first_run_complete`/`crash_reporting_enabled`
- Phase 49 decision: `POST /license/validate` returns 404 = not activated (first run), 200 = valid (redirect away from wizard), 403 = expired (re-activation)
- `ApiError` class at `@/lib/api-error` has `.status` and `.detail` properties — use `instanceof ApiError` in catch blocks
- `apiPost` from `@/lib/api` sends `Content-Type: application/json` — POST with no body is fine for `/license/validate`
- The project uses `"use client"` at top of all interactive pages
- `NEXT_PUBLIC_DESKTOP_MODE` is checked with `process.env.NEXT_PUBLIC_DESKTOP_MODE === 'true'` inside `useEffect` (never in SSR)
- Navbar hiding: `navbar-wrapper.tsx` has `hideNavbarPaths` array — currently `["/login", "/signup"]`
- Settings page at `frontend/src/app/settings/page.tsx` already has desktop version display gated on `NEXT_PUBLIC_DESKTOP_MODE`
- All Shadcn/UI components needed are already installed: Card, Button, Input, Alert, Switch, Progress
- `sonner` toast is used throughout the app for feedback
- `lucide-react` provides all icons needed

## Anti-Patterns to Avoid

- Do NOT put a first-run redirect guard in `layout.tsx` or app pages — only guard in `/setup` itself (redirect AWAY if already activated)
- Do NOT pre-fill input fields with redacted `***xxxx` hints — show "Key set (ends in ...xxxx)" as a label next to an empty field
- Do NOT send empty strings for optional API key fields — only include fields with non-empty trimmed values in the settings payload to avoid overwriting existing keys
- Do NOT use a stepper component library — `useState` with `currentStep` is sufficient for 3 steps
- Do NOT use `window.electronAPI` IPC — all communication is via HTTP `apiPost`/`apiGet` (Phase 49 decision)
- Do NOT block the app on first-run check — the wizard shows step 1 immediately; the redirect-away check runs async in background

```tsx
export default function SetupPage() {
```

**First-run guard useEffect (only runs when NOT in edit mode and in desktop mode):**

```tsx
  // First-run guard — redirect away if already activated (not in edit mode)
  useEffect(() => {
    if (isEditMode) {
      setCheckingLicense(false)
      return
    }
    if (process.env.NEXT_PUBLIC_DESKTOP_MODE !== "true") {
      setCheckingLicense(false)
      return
    }

    apiPost("/desktop/license/validate")
      .then(() => {
        // 200 = license valid = setup already done = redirect to app
        router.replace("/librarie")
      })
      .catch((err: unknown) => {
        if (err instanceof ApiError) {
          if (err.status === 404) {
            // Not activated — stay on wizard Step 1
            setCheckingLicense(false)
            return
          }
          if (err.status === 403) {
            // License expired — show re-activation in Step 1
            setLicenseError("Your license has expired or is invalid. Please re-activate.")
            setCheckingLicense(false)
            return
          }
        }
        // Network error or unexpected — assume first run, stay on wizard
        setCheckingLicense(false)
      })
  }, [isEditMode, router])
```

**Edit mode pre-fill useEffect:**

```tsx
  // Pre-fill current values in edit mode
  useEffect(() => {
    if (!isEditMode) return

    apiGet("/desktop/settings")
      .then((res) => res.json())
      .then((data: any) => {
        // Supabase URL is returned unredacted
        if (data.supabase_url) setSupabaseUrl(data.supabase_url)
        // Set hints for redacted keys (e.g., "***1234")
        if (data.supabase_key) setSupabaseHint(data.supabase_key)
        if (data.gemini_api_key) setGeminiHint(data.gemini_api_key)
        if (data.elevenlabs_api_key) setElevenlabsHint(data.elevenlabs_api_key)
        // Pre-fill crash reporting toggle
        if (data.crash_reporting_enabled) setCrashReporting(data.crash_reporting_enabled)
        // In edit mode, license is already valid — skip to step 2
        setLicenseValid(true)
        setCurrentStep(2)
      })
      .catch(() => {
        // Failed to load — start from step 1
      })
  }, [isEditMode])
```

**License activation handler:**

```tsx
  const handleActivate = async () => {
    if (!licenseKey.trim()) {
      setLicenseError("Please enter a license key")
      return
    }
    setActivating(true)
    setLicenseError(null)
    try {
      await apiPost("/desktop/license/activate", { license_key: licenseKey.trim() })
      setLicenseValid(true)
      toast.success("License activated successfully!")
      // Auto-advance to step 2 after short delay for user to see success
      setTimeout(() => setCurrentStep(2), 800)
    } catch (err: unknown) {
      const msg = err instanceof ApiError
        ? err.detail || "Activation failed"
        : "Network error — please check your internet connection"
      setLicenseError(msg)
    } finally {
      setActivating(false)
    }
  }
```

**Connection test handler:**

```tsx
  const testConnection = async (service: string) => {
    const setStatus = service === "supabase" ? setSupabaseStatus
      : service === "gemini" ? setGeminiStatus
      : setElevenlabsStatus

    const url = service === "supabase" ? supabaseUrl : ""
    const key = service === "supabase" ? supabaseKey
      : service === "gemini" ? geminiKey
      : elevenlabsKey

    if (!key.trim() && service !== "supabase") {
      toast.warning(`Please enter a ${service} API key first`)
      return
    }
    if (service === "supabase" && (!supabaseUrl.trim() || !supabaseKey.trim())) {
      toast.warning("Please enter both Supabase URL and key")
      return
    }

    setStatus("testing")
    try {
      await apiPost("/desktop/test-connection", {
        service,
        url: url.trim(),
        key: key.trim(),
      })
      setStatus("ok")
      toast.success(`${service.charAt(0).toUpperCase() + service.slice(1)} connected!`)
    } catch (err: unknown) {
      setStatus("error")
      const msg = err instanceof ApiError ? err.detail : "Connection failed"
      toast.error(msg)
    }
  }
```

```tsx
      router.replace("/librarie")
    } catch {
      toast.error("Failed to save settings. Please try again.")
    } finally {
      setFinishing(false)
    }
  }
```

**Critical implementation details:**
1. `ApiError` imported from `@/lib/api-error` — use `instanceof ApiError` in catch blocks
2. `apiGet` used for `GET /settings` in edit mode (not `apiPost`)
3. `apiPost` used for `POST /license/validate` with NO body argument — FastAPI accepts POST with empty body
4. Only non-empty trimmed values included in settings payload on finish — avoids overwriting existing keys with empty strings
5. `first-run/complete` only called when NOT in edit mode — edit mode skips the first-run flag
6. Progress bar shows step completion: 0%, 33%, 66% (based on `(currentStep - 1) / 3`)
7. In edit mode, license step is skipped (currentStep starts at 2, Back button disabled on step 2)
8. The `checkingLicense` loading state prevents the wizard from briefly showing then redirecting
9. Enter key triggers activation on step 1 license input
