**What:** Add `sentry_dsn` to Settings, update crash_reporter to accept it, and initialize Sentry for all deployment modes (not just desktop).

**Files:** `app/config.py`, `app/services/crash_reporter.py`, `app/main.py`, `requirements.txt`

**Steps:**

1. In `app/config.py` Settings class, add:
   ```python
   sentry_dsn: str = ""  # Sentry DSN for error reporting (all modes)
   ```

2. In `app/services/crash_reporter.py`:
   - Remove the hardcoded `SENTRY_DSN = ""` module-level constant
   - The `init_sentry(dsn, enabled)` function signature stays the same — callers pass the DSN

3. In `app/main.py`, replace the desktop-only Sentry block (lines 216-228) with a two-path init:
   ```python
   # Sentry crash reporting
   # Path 1: SENTRY_DSN env var — works in all modes (production server, dev, desktop)
   if settings.sentry_dsn:
       from app.services.crash_reporter import init_sentry
       init_sentry(dsn=settings.sentry_dsn, enabled=True)
   # Path 2: Desktop-mode config.json opt-in (legacy, no SENTRY_DSN needed if DSN is hardcoded)
   elif settings.desktop_mode:
       from app.services.crash_reporter import init_sentry, SENTRY_DSN
       import json as _json
       _config_file = settings.base_dir / "config.json"
       _crash_enabled = False
       if _config_file.exists():
           try:
               _cfg = _json.loads(_config_file.read_text(encoding="utf-8"))
               _crash_enabled = _cfg.get("crash_reporting_enabled", False)
           except Exception:
               pass
       if SENTRY_DSN:
           init_sentry(dsn=SENTRY_DSN, enabled=_crash_enabled)
   ```

4. In `requirements.txt`, ensure `sentry-sdk[fastapi]` is listed (it may already be there from Phase 51 desktop crash reporting). If already present, no change needed. If not, add with a pinned version:
   ```
   sentry-sdk[fastapi]==2.19.2
   ```

5. In `app/services/crash_reporter.py`, update `init_sentry` to also install the FastAPI integration when available:
   ```python
   # Inside init_sentry, after sentry_sdk.init():
   # The [fastapi] extra auto-registers the ASGI integration
   # which captures unhandled exceptions from route handlers
   ```
   Note: `sentry-sdk[fastapi]` automatically registers the FastAPIIntegration when imported. No manual integration registration needed if using sentry-sdk >= 2.x.

<task id="2" title="Extend health check with Supabase connectivity and granular status">

**What:** Add Supabase connectivity test to the health endpoint and return individual status strings for each dependency.

**Files:** `app/models.py`, `app/api/routes.py`

**Steps:**

1. In `app/models.py`, update HealthResponse to add granular status fields while keeping backward compatibility:
   ```python
   class HealthResponse(BaseModel):
       """Health check response with individual dependency status."""
       status: str  # "ok", "degraded", or "unhealthy"
       version: str
       ffmpeg_available: bool  # kept for backward compat
       redis_available: bool   # kept for backward compat
       # Granular dependency status (MON-02)
       supabase_status: str = "unknown"   # "ok" | "unavailable" | "unknown"
       ffmpeg_status: str = "unknown"     # "ok" | "unavailable"
       redis_status: str = "unknown"      # "ok" | "unavailable"
   ```

2. In `app/api/routes.py`, update `health_check()` to test Supabase:
   ```python
   @router.get("/health", response_model=HealthResponse)
   async def health_check():
       """Check health of all dependencies."""
       # Check FFmpeg
       ffmpeg_ok = False
       try:
           result = await asyncio.to_thread(subprocess.run, ["ffmpeg", "-version"], capture_output=True, timeout=5)
           ffmpeg_ok = result.returncode == 0
       except Exception:
           pass

       # Check Supabase connectivity
       supabase_ok = False
       try:
           from app.db import get_supabase
           supabase = get_supabase()
           if supabase:
               # Lightweight query: count with limit 0 — no data transfer
               result = await asyncio.to_thread(
                   lambda: supabase.table("editai_projects").select("id", count="exact").limit(0).execute()
               )
               supabase_ok = True  # If we got here without exception, DB is reachable
       except Exception:
           pass

       # Check Redis (optional)
       redis_ok = False
       try:
           import redis
           settings = get_settings()
           r = redis.from_url(settings.redis_url)
           try:
               r.ping()
               redis_ok = True
           finally:
               r.close()
       except Exception:
           pass

       # Determine overall status
       # "ok" = Supabase AND FFmpeg are up (Redis is optional)
       # "degraded" = one of Supabase/FFmpeg is down
       # "unhealthy" = both Supabase AND FFmpeg are down
       if supabase_ok and ffmpeg_ok:
           overall = "ok"
       elif not supabase_ok and not ffmpeg_ok:
           overall = "unhealthy"
       else:
           overall = "degraded"

       return HealthResponse(
           status=overall,
           version=APP_VERSION,
           ffmpeg_available=ffmpeg_ok,
           redis_available=redis_ok,
           supabase_status="ok" if supabase_ok else "unavailable",
           ffmpeg_status="ok" if ffmpeg_ok else "unavailable",
           redis_status="ok" if redis_ok else "unavailable",
       )
   ```

**Verify:**

- With valid Supabase credentials, `supabase_status` is "ok" and `status` is "ok" (assuming FFmpeg works)
- The response still includes `ffmpeg_available` and `redis_available` booleans for backward compat
- Existing test `test_health_returns_200` in `tests/test_api_routes.py` still passes (it checks for "status" key)
