# Plan 58-03: Unify Assembly Jobs into JobStorage

<objective>
After this plan, assembly jobs appear in the same job status endpoint (`GET /api/v1/jobs/{job_id}`) as video processing jobs. No separate polling path is needed. The `/assembly/status/{job_id}` endpoint remains as an alias for backward compatibility but internally delegates to JobStorage.

Purpose: Eliminate the fragmented job storage — one unified system for all job types.
Output: Assembly jobs stored in JobStorage with backward-compatible status endpoint.
</objective>

**Current state:**
- Video processing jobs use `JobStorage` (Supabase `jobs` table + in-memory fallback)
- Assembly jobs use their own `_assembly_jobs` in-memory dict + separate DB helpers writing to `editai_assembly_jobs` table
- `/api/v1/jobs/{job_id}` only queries JobStorage — assembly jobs are invisible there
- `/api/v1/assembly/status/{job_id}` only queries `_assembly_jobs` — a separate polling path
- Product generate jobs already use JobStorage (see `product_generate_routes.py`)

**Gap:** Assembly jobs are the only job type with their own storage system. This plan unifies them into JobStorage while keeping backward compatibility.
</context>

  <files>

  <action>
Modify `app/api/assembly_routes.py` to use `JobStorage` alongside the existing custom storage (dual-write for backward compatibility):

1. **Add import** at the top:
   ```python
   from app.services.job_storage import get_job_storage
   ```

2. **In `render_assembly()` endpoint:** After the existing `_assembly_jobs[job_id] = {...}` line, also create job in JobStorage:
   ```python
   job_storage = get_job_storage()
   job_data = {
       "job_id": job_id,
       "job_type": "assembly",
       "status": "processing",
       "progress": "Initializing assembly",
       "progress_percentage": 0,
       "current_step": "Initializing assembly",
       "profile_id": profile.profile_id,
       "started_at": datetime.now(timezone.utc).isoformat(),
   }
   job_storage.create_job(job_data, profile_id=profile.profile_id)
   ```
   Keep the existing `_assembly_jobs` dict write AND `_db_create_assembly_job()` call for backward compatibility.

3. **In the `do_assembly()` background task:** At each progress update point, add a `get_job_storage().update_job()` call alongside the existing `_assembly_jobs[job_id]` dict mutation and `_db_update_assembly_job()` call:

   Progress update pattern:
   ```python
   _assembly_jobs[job_id]["progress"] = progress_pct
   _assembly_jobs[job_id]["current_step"] = step_text
   get_job_storage().update_job(job_id, {
       "status": "processing",
       "progress": step_text,
       "progress_percentage": progress_pct,
       "current_step": step_text,
   })
   _db_update_assembly_job(job_id, _assembly_jobs[job_id])
   ```

   On completion:
   ```python
   _assembly_jobs[job_id]["status"] = "completed"
   _assembly_jobs[job_id]["final_video_path"] = final_path
   get_job_storage().update_job(job_id, {
       "status": "completed",
       "progress": "Assembly complete",
       "progress_percentage": 100,
       "current_step": "Complete",
       "final_video_path": final_path,
   })
   _db_update_assembly_job(job_id, _assembly_jobs[job_id])
   ```

   On failure:
   ```python
   _assembly_jobs[job_id]["status"] = "failed"
   _assembly_jobs[job_id]["error"] = str(e)
   get_job_storage().update_job(job_id, {
       "status": "failed",
       "progress": "Assembly failed",
       "error": str(e),
   })
   _db_update_assembly_job(job_id, _assembly_jobs[job_id])
   ```

4. **In `get_assembly_status()` endpoint:** Change to read from JobStorage first, then fall back to `_assembly_jobs` and legacy DB:

   ```python
   async def get_assembly_status(job_id: str, ...):
       # Try JobStorage first (unified)
       job_storage = get_job_storage()
       job_data = job_storage.get_job(job_id)

       if job_data and job_data.get("job_type") == "assembly":
           error_msg = None
           if job_data.get("error"):
               error_msg = "Processing failed. Check server logs for details."
           return AssemblyStatusResponse(
               status=job_data.get("status", "processing"),
               progress=job_data.get("progress_percentage", 0),
               current_step=job_data.get("current_step", ""),
               final_video_path=job_data.get("final_video_path"),
               error=error_msg,
           )

       # Fallback to legacy _assembly_jobs dict and DB
       if job_id not in _assembly_jobs:
           loaded = _db_load_assembly_job(job_id)
           if not loaded:
               raise HTTPException(status_code=404, detail="Job not found")
       # ... existing legacy response code ...
   ```
