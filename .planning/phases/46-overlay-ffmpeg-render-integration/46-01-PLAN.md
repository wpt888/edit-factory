Create `app/services/overlay_renderer.py` with two core functions:

1. **`generate_interstitial_clip(image_url_or_path, output_path, duration, animation, ken_burns_direction, width=1080, height=1920, fps=30)`**
   - Downloads image from URL (or uses local path) to a temp file if needed
   - For `animation="static"`: scale image to fill 1080x1920 (force_original_aspect_ratio=increase, crop center), create video with `-loop 1 -t {duration}`
   - For `animation="kenburns"`: pre-scale image to 4x (4320x7680) for smooth motion, apply zoompan filter
     - Reuse the zoompan math from `product_video_compositor.py`: `_calculate_zoompan_params` and `_build_zoompan_filter` approach
     - Map `ken_burns_direction` values: "zoom-in" -> zoom 1.0->1.5, "zoom-out" -> zoom 1.5->1.0, "pan-left" -> x pans left, "pan-right" -> x pans right
   - Output: H.264, yuv420p, no audio, portrait 1080x1920
   - Returns the output_path on success

2. **`apply_pip_overlay(video_path, image_url_or_path, output_path, position, size, animation, duration=None, width=1080, height=1920)`**
   - Downloads image if URL, otherwise uses local path
   - Map `size` to pixel dimensions: "small" = 150x150, "medium" = 200x200, "large" = 280x280
   - Map `position` to FFmpeg overlay coordinates:
     - "top-left": `x=40:y=200` (below TikTok safe zone)
     - "top-right": `x=W-w-40:y=200`
     - "bottom-left": `x=40:y=H-h-250` (above bottom UI)
     - "bottom-right": `x=W-w-40:y=H-h-250`
   - Scale image to target size with rounded corners (use `format=rgba,geq` for rounded mask or just skip rounding for simplicity — square PiP is fine)
   - For `animation="static"`: overlay for entire segment duration
   - For `animation="kenburns"`: apply zoompan to the PiP image before overlaying (pre-scale 2x, zoompan, then scale down to pip size)
   - For `animation="fade"`: use `overlay=....:enable='between(t,0.5,{dur-0.5})'` with fade in/out using `fade=t=in:st=0:d=0.5,fade=t=out:st={dur-1}:d=0.5`
   - Uses `-filter_complex` with labeled streams: `[0:v]` (video), `[1:v]` (pip image) -> scale pip -> optional animation -> overlay
   - If `animation="static"`: simple `[1:v]scale={size}:{size}[pip];[0:v][pip]overlay={x}:{y}[out]`
   - Returns output_path on success, returns original video_path on failure (graceful degradation)

3. **Helper: `_download_image(url_or_path, temp_dir)`**
   - If path exists locally, return it
   - If URL (starts with http), download via httpx/aiohttp to temp_dir, return path
   - Use `httpx` (already in requirements) with 10s timeout
   - Return None on failure (graceful degradation)

All functions should be async-compatible (use `asyncio.to_thread` for subprocess calls like assembly_service does). Include proper logging. On any FFmpeg failure, log error and return None (never crash the render pipeline).

```typescript
// Build pip_overlays: segment_id -> { image_url, position, size, animation }
const pipOverlays: Record<string, { image_url: string; position: string; size: string; animation: string }> = {};
for (const idx of Array.from(selectedVariants)) {
  const preview = previews[idx];
  if (!preview?.matches) continue;
  for (const match of preview.matches) {
    if (!match.segment_id) continue;
    const assoc = associations[match.segment_id];
    if (!assoc?.pip_config?.enabled) continue;
    // Use first selected image, or product_image fallback
    const imageUrl = assoc.selected_image_urls?.[0] || assoc.product_image;
    if (!imageUrl) continue;
    pipOverlays[match.segment_id] = {
      image_url: imageUrl,
      position: assoc.pip_config.position,
      size: assoc.pip_config.size,
      animation: assoc.pip_config.animation,
    };
  }
}
```
