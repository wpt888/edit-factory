```python
def _get_fernet():
    """Get Fernet instance using ELEVENLABS_ENCRYPTION_KEY or derive from SUPABASE_KEY."""
    settings = get_settings()
    key = getattr(settings, 'elevenlabs_encryption_key', None)
    if not key:
        base_key = settings.supabase_key or "edit-factory-default-key"
        derived = hashlib.sha256(base_key.encode()).digest()
        key = base64.urlsafe_b64encode(derived)
    return Fernet(key)
```

  <files>app/services/key_vault.py</files>
  <action>
Create `app/services/key_vault.py` — a service that stores and retrieves API keys from an encrypted JSON file on disk.

Implementation details:
- Use the SAME Fernet key derivation pattern as `elevenlabs_account_manager.py` (derive from SUPABASE_KEY or ELEVENLABS_ENCRYPTION_KEY setting, or use a machine-specific fallback).
- For the machine-specific fallback (when no SUPABASE_KEY exists — i.e., SQLite-only desktop mode), derive the Fernet key from a combination of: the machine hostname + a salt stored in a separate `vault_salt.bin` file (auto-generated on first use via `os.urandom(32)`). This ensures the encrypted file is tied to the machine.
- Storage file: `{base_dir}/keys.vault` (a JSON structure where each key name maps to a Fernet-encrypted value string).
- Public API:
  ```python
  class KeyVault:
      def __init__(self, base_dir: Path): ...
      def store_key(self, name: str, plaintext: str) -> None: ...
      def get_key(self, name: str) -> Optional[str]: ...
      def get_key_hint(self, name: str) -> str: ...  # Returns "***last4" or ""
      def has_key(self, name: str) -> bool: ...
      def delete_key(self, name: str) -> None: ...
      def list_keys(self) -> List[str]: ...  # Returns key names (not values)

  def get_key_vault() -> KeyVault:  # Singleton factory
  ```
- Key names to support: `gemini_api_key`, `elevenlabs_api_key`, `elevenlabs_voice_id`, `supabase_url`, `supabase_key`.
- The `store_key` method encrypts the value with Fernet before writing to disk.
- The `get_key` method reads the encrypted value, decrypts, and returns plaintext.
- The `get_key_hint` method returns `"***{last4chars}"` without decrypting the full key (store a separate `_hints` dict in the vault file with redacted values for display purposes).
- Handle backward compatibility: if `config.json` has plaintext keys from before this change, `get_key` should also check config.json as fallback and migrate those keys to the vault on first access.
- Thread-safe: use a `threading.Lock` around file read/write operations.
