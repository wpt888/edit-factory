## Goal

Display the app version in the Settings page footer when running in desktop mode, by calling `GET /api/v1/desktop/version` and rendering the version string.

## Context

- Plan 49-01 creates the `GET /api/v1/desktop/version` endpoint that returns `{"version": "0.1.0"}`
- The Settings page is at `frontend/src/app/settings/page.tsx` (~1011 lines)
- The page uses `apiGetWithRetry` and `apiPost` from `@/lib/api` for API calls
- Desktop mode is detected via `process.env.NEXT_PUBLIC_DESKTOP_MODE === 'true'` (environment variable set by Electron)
- The page's return JSX ends with a Save button `<div>` (lines 991-1008), followed by closing `</div>` and `}` (lines 1009-1011)
- The version display should appear AFTER the Save button div but BEFORE the final closing `</div>`

## Tasks

<task id="1">
<title>Add version state and fetch to Settings page</title>
<details>
In `frontend/src/app/settings/page.tsx`:

1. **Add version state** — near the other `useState` declarations at the top of the component function, add:

```typescript
const [appVersion, setAppVersion] = useState<string | null>(null)
```

2. **Add version fetch useEffect** — add a new `useEffect` after the existing ones (but before the handler functions):

```typescript
useEffect(() => {
  if (process.env.NEXT_PUBLIC_DESKTOP_MODE !== 'true') return
  apiGetWithRetry('/desktop/version')
    .then((data: any) => setAppVersion(data.version))
    .catch(() => {}) // Non-critical — silently ignore errors
}, [])
```

3. **Add version display JSX** — insert AFTER the Save button `<div className="flex justify-end">...</div>` block (after line 1008) and BEFORE the final closing `</div>` (line 1009):

```tsx
{appVersion && (
  <div className="text-center text-xs text-muted-foreground mt-8 pb-4">
    Edit Factory v{appVersion}
  </div>
)}
```

**Result:** When `NEXT_PUBLIC_DESKTOP_MODE=true`, the Settings page shows "Edit Factory v0.1.0" at the bottom. When not in desktop mode (or if the API call fails), nothing is shown.
</details>
</task>

## Verification

After execution, verify:

1. **UPDT-06 check:** The Settings page component has `appVersion` state, a `useEffect` that calls `/desktop/version` when `NEXT_PUBLIC_DESKTOP_MODE === 'true'`, and renders the version string in a footer element
2. **Non-desktop check:** The version fetch is gated behind the `NEXT_PUBLIC_DESKTOP_MODE` check — in normal dev mode, no API call is made and no version is displayed
3. **Error resilience:** The `.catch(() => {})` ensures a failed version fetch does not break the Settings page
