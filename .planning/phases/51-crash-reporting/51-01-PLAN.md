## Goal

Create `app/services/crash_reporter.py` with conditional Sentry initialization, PII scrubbing, and a runtime toggle flag. Wire it into `app/main.py` (desktop-mode only) and add a `POST /desktop/crash-reporting` endpoint to `desktop_routes.py`.

## Context

- Research: `.planning/phases/51-crash-reporting/51-RESEARCH.md`
- `sentry_sdk.init()` MUST be called ONCE, AFTER `app = FastAPI(...)` — calling it twice is undefined behavior (Sentry issue #3059)
- Runtime toggle uses a module-level `_crash_reporting_enabled` flag checked in `before_send` — when False, events are dropped before leaving the machine
- `EventScrubber` with extended denylist handles PII scrubbing (UPDT-04)
- DSN is a public endpoint, safe to embed in source code — standard Sentry practice
- `config.json` already stores `crash_reporting_enabled` (written by Setup Wizard Phase 50)
- `GET /desktop/settings` already returns `crash_reporting_enabled` (see `desktop_routes.py` line 102)

<task id="2">
<title>Create app/services/crash_reporter.py</title>
<details>
Create NEW file `app/services/crash_reporter.py` with:

1. Module-level mutable flag `_crash_reporting_enabled: bool = False`

2. Custom denylist extending `DEFAULT_DENYLIST`:
   - Add: `gemini_api_key`, `supabase_key`, `supabase_url`, `elevenlabs_api_key`, `anthropic_api_key`, `license_key`, `instance_id`

3. `_before_send(event, hint)` function:
   - If `_crash_reporting_enabled` is False, return None (drop event)
   - Otherwise return event unchanged

4. `init_sentry(dsn: str, enabled: bool) -> None` function:
   - Set `_crash_reporting_enabled = enabled`
   - If `not enabled or not dsn`: return immediately (no-op — no `sentry_sdk.init()` call)
   - Otherwise call `sentry_sdk.init()` with:
     - `dsn=dsn`
     - `send_default_pii=False`
     - `include_local_variables=True`
     - `event_scrubber=EventScrubber(denylist=_CUSTOM_DENYLIST, recursive=True)`
     - `before_send=_before_send`
     - `traces_sample_rate=0.0` (error-only, no performance tracing)

5. `set_crash_reporting(enabled: bool) -> None` function:
   - Update `_crash_reporting_enabled` global flag
   - This is the ONLY mechanism for runtime toggle — never call `init_sentry()` again

6. `is_enabled() -> bool` function:
   - Return current value of `_crash_reporting_enabled`

Use a placeholder DSN constant at module level:
```python
# Replace with actual Sentry DSN when Sentry project is created
SENTRY_DSN = ""
```

Follow the exact pattern from 51-RESEARCH.md Pattern 1.
</details>
</task>

```python
# Sentry crash reporting (desktop-mode only, opt-in)
if settings.desktop_mode:
    from app.services.crash_reporter import init_sentry, SENTRY_DSN
    import json as _json
    _config_file = settings.base_dir / "config.json"
    _crash_enabled = False
    if _config_file.exists():
        try:
            _cfg = _json.loads(_config_file.read_text(encoding="utf-8"))
            _crash_enabled = _cfg.get("crash_reporting_enabled", False)
        except Exception:
            pass
    init_sentry(dsn=SENTRY_DSN, enabled=_crash_enabled)
```

CRITICAL placement: This MUST be AFTER `app = FastAPI(...)` (line 129) and AFTER middleware. The Sentry FastAPI integration instruments the app object during init.

Do NOT modify the existing `if settings.desktop_mode:` block at line 184 that imports desktop_routes.
</details>

<task id="4">
<title>Add POST /desktop/crash-reporting endpoint to desktop_routes.py</title>
<details>
In `app/api/desktop_routes.py`, add a new endpoint after the existing `save_desktop_settings` endpoint (after line 192):

```python
@router.post("/crash-reporting")
async def set_crash_reporting_toggle(body: dict):
    """Toggle crash reporting at runtime. Takes immediate effect without restart."""
    enabled = bool(body.get("enabled", False))
    # Update in-memory flag (immediate effect via before_send)
    from app.services.crash_reporter import set_crash_reporting
    set_crash_reporting(enabled)
    # Persist to config.json
    settings = get_settings()
    config_file = settings.base_dir / "config.json"
    existing = _read_config(config_file)
    existing["crash_reporting_enabled"] = enabled
    config_file.write_text(json.dumps(existing, indent=2), encoding="utf-8")
    logger.info("Crash reporting toggled: %s", "enabled" if enabled else "disabled")
    return {"crash_reporting_enabled": enabled}
```

The `_read_config` helper and `json` import already exist in this file. The `set_crash_reporting` import is lazy (inside function body) to avoid import-time issues when not in desktop mode.
</details>
</task>

## must_haves

Derived from phase goal: "Users who opted in during setup have crashes automatically reported to Sentry -- with API keys and file paths scrubbed before any data leaves the machine"

- [ ] `sentry_sdk.init()` called ONLY when `crash_reporting_enabled` is True in config.json AND `desktop_mode` is True — no Sentry in dev/web mode (UPDT-03)
- [ ] `before_send` hook drops all events when `_crash_reporting_enabled` is False — runtime toggle without restart (UPDT-03)
- [ ] `EventScrubber` with extended denylist covering `gemini_api_key`, `supabase_key`, `supabase_url`, `elevenlabs_api_key`, `anthropic_api_key`, `license_key`, `instance_id` (UPDT-04)
- [ ] `send_default_pii=False` to prevent IP/header leakage (UPDT-04)
- [ ] `POST /desktop/crash-reporting` endpoint updates both in-memory flag AND persists to config.json (UPDT-03)
