## Goal

Create the Next.js standalone postbuild script that copies static assets and public files into the standalone output directory, and create a placeholder app icon for the Electron tray. These are prerequisites for the Electron shell to serve the frontend correctly.

## Context

Current state:
- `frontend/next.config.ts` already has `output: "standalone"` (line 5)
- `frontend/package.json` build script is `"build": "next build"` — does not copy static assets
- Next.js standalone output at `frontend/.next/standalone/` does NOT include `.next/static/` or `public/` (documented pitfall in research)
- Without these copies, the frontend loads but all CSS/JS chunks return 404

Research file: `.planning/phases/48-electron-shell/48-RESEARCH.md` — see Pitfall 6 (static assets missing) and Pattern: Next.js Standalone Post-Build Copy.

SHELL-01 requires Electron to spawn Next.js standalone as a child process. That child process needs the standalone output to be complete (static + public copied in).

## Tasks

<task id="A1">
<title>Create frontend/scripts/postbuild.js for standalone asset copy</title>
<details>
Create `frontend/scripts/postbuild.js` using Node.js `fs` module (cross-platform, works on Windows and WSL):

```javascript
// frontend/scripts/postbuild.js
// Copies static assets into Next.js standalone output.
// Required for standalone server.js to serve CSS, JS, and public files.
const fs = require('fs');
const path = require('path');

function copyDir(src, dest) {
  fs.mkdirSync(dest, { recursive: true });
  for (const entry of fs.readdirSync(src, { withFileTypes: true })) {
    const s = path.join(src, entry.name);
    const d = path.join(dest, entry.name);
    if (entry.isDirectory()) copyDir(s, d);
    else fs.copyFileSync(s, d);
  }
}

const base = path.join(__dirname, '..');
const standaloneDir = path.join(base, '.next', 'standalone');

// Check standalone output exists
if (!fs.existsSync(standaloneDir)) {
  console.error('ERROR: .next/standalone/ not found. Run "next build" first.');
  process.exit(1);
}

// Copy .next/static -> .next/standalone/.next/static
const staticSrc = path.join(base, '.next', 'static');
const staticDest = path.join(standaloneDir, '.next', 'static');
if (fs.existsSync(staticSrc)) {
  copyDir(staticSrc, staticDest);
  console.log('Copied .next/static -> standalone/.next/static');
} else {
  console.warn('WARNING: .next/static/ not found — skipping');
}

// Copy public -> .next/standalone/public
const publicSrc = path.join(base, 'public');
const publicDest = path.join(standaloneDir, 'public');
if (fs.existsSync(publicSrc)) {
  copyDir(publicSrc, publicDest);
  console.log('Copied public/ -> standalone/public/');
} else {
  console.warn('WARNING: public/ not found — skipping');
}

console.log('Standalone assets copied successfully.');
```

This uses only Node.js stdlib — no dependencies needed. Works on Windows cmd, PowerShell, and WSL/Linux.
</details>
</task>

<task id="A2">
<title>Update frontend/package.json build script to include postbuild</title>
<details>
In `frontend/package.json`, update the `build` script from:

```json
"build": "next build"
```

to:

```json
"build": "next build && node scripts/postbuild.js"
```

This ensures every `npm run build` automatically copies static assets into the standalone directory. The `&&` ensures postbuild only runs if `next build` succeeds.
</details>
</task>

<task id="A3">
<title>Create placeholder app icon for Electron tray</title>
<details>
Create the directory `electron/build/` and place a minimal `icon.ico` file there.

Generate a minimal valid ICO file using Node.js. Create a script `electron/build/generate-icon.js`:

```javascript
// Generates a minimal 16x16 ICO file as placeholder.
// Replace with real branding icon before release (Phase 52).
const fs = require('fs');
const path = require('path');

// Minimal 16x16 32-bit RGBA ICO: dark blue square (#1a1a2e)
// ICO header (6 bytes) + directory entry (16 bytes) + BMP header (40 bytes) + pixel data (16*16*4 = 1024 bytes)
const width = 16;
const height = 16;
const bpp = 32;
const pixelDataSize = width * height * (bpp / 8);
const bmpHeaderSize = 40;
const imageSize = bmpHeaderSize + pixelDataSize;

const buf = Buffer.alloc(6 + 16 + imageSize);
let offset = 0;

// ICO header
buf.writeUInt16LE(0, offset); offset += 2; // reserved
buf.writeUInt16LE(1, offset); offset += 2; // type: 1 = ICO
buf.writeUInt16LE(1, offset); offset += 2; // count: 1 image

// Directory entry
buf.writeUInt8(width, offset); offset += 1;
buf.writeUInt8(height, offset); offset += 1;
buf.writeUInt8(0, offset); offset += 1; // palette
buf.writeUInt8(0, offset); offset += 1; // reserved
buf.writeUInt16LE(1, offset); offset += 2; // color planes
buf.writeUInt16LE(bpp, offset); offset += 2; // bits per pixel
buf.writeUInt32LE(imageSize, offset); offset += 4; // image size
buf.writeUInt32LE(6 + 16, offset); offset += 4; // image offset

// BMP info header
buf.writeUInt32LE(bmpHeaderSize, offset); offset += 4;
buf.writeInt32LE(width, offset); offset += 4;
buf.writeInt32LE(height * 2, offset); offset += 4; // height doubled for ICO
buf.writeUInt16LE(1, offset); offset += 2; // planes
buf.writeUInt16LE(bpp, offset); offset += 2;
buf.writeUInt32LE(0, offset); offset += 4; // compression
buf.writeUInt32LE(pixelDataSize, offset); offset += 4;
buf.writeInt32LE(0, offset); offset += 4; // x ppm
buf.writeInt32LE(0, offset); offset += 4; // y ppm
buf.writeUInt32LE(0, offset); offset += 4; // colors used
buf.writeUInt32LE(0, offset); offset += 4; // important colors

// Pixel data: BGRA, bottom-up, dark blue #1a1a2e
for (let i = 0; i < width * height; i++) {
  buf.writeUInt8(0x2e, offset); offset += 1; // B
  buf.writeUInt8(0x1a, offset); offset += 1; // G
  buf.writeUInt8(0x1a, offset); offset += 1; // R
  buf.writeUInt8(0xff, offset); offset += 1; // A
}

fs.writeFileSync(path.join(__dirname, 'icon.ico'), buf);
console.log('Generated placeholder icon.ico');
```

Run it once: `cd electron/build && node generate-icon.js`

This produces a valid 16x16 ICO file. A production-quality multi-size ICO (16/32/48/256) will replace this in Phase 52.

**Alternative approach (simpler):** If generating a valid ICO is too complex, create a 1x1 PNG and save it as `icon.png`. Electron's Tray accepts PNG on Windows (with slightly reduced quality). The code in Plan B should handle both: `const iconPath = path.join(__dirname, '..', 'build', fs.existsSync(...ico) ? 'icon.ico' : 'icon.png')`.

**Recommended:** Use the ICO generator approach above. It produces a valid, committed file with no external tools needed.
</details>

## Verification

```
PASS criteria:
1. `frontend/scripts/postbuild.js` exists and contains copyDir function
2. `frontend/package.json` build script includes `&& node scripts/postbuild.js`
3. `electron/build/icon.ico` exists and is a valid ICO file (non-zero size)
4. Running `cd frontend && npm run build` succeeds (builds Next.js + copies static assets)
5. After build, `frontend/.next/standalone/.next/static/` directory exists with chunk files
6. After build, `frontend/.next/standalone/public/` directory exists (if public/ has content)
```

## must_haves

- `frontend/scripts/postbuild.js` copies `.next/static` into `.next/standalone/.next/static`
- `frontend/scripts/postbuild.js` copies `public/` into `.next/standalone/public/`
- `frontend/package.json` build script chains postbuild after `next build`
- `electron/build/icon.ico` exists as a valid ICO file for the tray icon
- Postbuild script uses Node.js fs only (no Unix-specific commands like cp)
