  <action>
Create a reusable dialog component for searching and selecting a catalog product to associate with a segment.

**Props interface:**
```typescript
interface ProductPickerDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  segmentId: string;
  onProductSelected: (association: AssociationResponse) => void;
}
```

Where `AssociationResponse` matches the backend response shape:
```typescript
interface AssociationResponse {
  id: string;
  segment_id: string;
  catalog_product_id: string;
  selected_image_urls: string[];
  pip_config: Record<string, unknown> | null;
  slide_config: Record<string, unknown> | null;
  created_at: string;
  updated_at: string;
  product_title: string | null;
  product_image: string | null;
  product_brand: string | null;
}
```

**Component behavior:**

1. On dialog open, fetch filter options via `apiGet("/catalog/products/filters")` to populate brand and category dropdowns.
2. Fetch products via `apiGet("/catalog/products?search=...&brand=...&category=...&page=...&page_size=20")` — debounce search input by 300ms using a `useRef` timer pattern (no external debounce library).
3. Display products in a scrollable grid of cards. Each card shows: product thumbnail (`image_link`), title, brand badge, price (`raw_price_str`), sale badge if `is_on_sale`, variant count badge if `variant_count > 1`.
4. Pagination: show current page, total pages, prev/next buttons at the bottom — follow the exact pattern from `products/page.tsx`.
5. On card click: call `apiPost("/associations", { segment_id: segmentId, catalog_product_id: product.id })`. On success, call `onProductSelected(response)` and close the dialog. On error, show toast via sonner.
6. Loading states: show `Loader2` spinner during fetch. Show `EmptyState` component (from `@/components/empty-state`) when no products match.

**UI layout:**
- Use Shadcn Dialog (DialogContent with `className="max-w-3xl max-h-[80vh]"` for large modal).
- Top section: search Input + brand Select + category Select in a flex row.
- Content section: ScrollArea wrapping a grid of product cards (`grid grid-cols-2 sm:grid-cols-3 gap-3`).
- Bottom section: pagination controls.

**Imports to use:**
- `Dialog, DialogContent, DialogHeader, DialogTitle` from `@/components/ui/dialog`
- `Button` from `@/components/ui/button`
- `Input` from `@/components/ui/input`
- `Select, SelectContent, SelectItem, SelectTrigger, SelectValue` from `@/components/ui/select`
- `ScrollArea` from `@/components/ui/scroll-area`
- `Badge` from `@/components/ui/badge`

- `apiGet, apiPost, handleApiError` from `@/lib/api`
- `toast` from `sonner`
- `Search, Loader2, Package, ChevronLeft, ChevronRight` from `lucide-react`
- `EmptyState` from `@/components/empty-state`

Follow the existing dialog patterns from `create-feed-dialog.tsx` (controlled open/onOpenChange, loading state, toast on error, callback on success).

  <action>
Create a reusable dialog component for viewing and selecting product gallery images on an existing association.

**Props interface:**
```typescript
interface ImagePickerDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  associationId: string;
  catalogProductId: string;
  currentSelectedUrls: string[];
  productTitle: string | null;
  onImagesUpdated: (updatedAssociation: AssociationResponse) => void;
}
```

Reuse the same `AssociationResponse` type from Task 1 (export it from a shared location or define inline — keep it simple, inline is fine since Phase 34 will consolidate types).

**Component behavior:**

1. On dialog open, fetch all images via `apiGet("/catalog/products/${catalogProductId}/images")` which returns `{ product_id, images: string[] }`.
2. Initialize a local `selectedUrls` state from `currentSelectedUrls` prop. Use a `Set<string>` for O(1) toggle lookups.
3. Display images in a responsive grid (`grid grid-cols-2 sm:grid-cols-3 md:grid-cols-4 gap-3`). Each image cell:
   - Shows the image as a square thumbnail with `object-cover` and rounded corners.
   - Overlay a checkmark icon (Lucide `Check`) in the top-right corner when selected, with a green tinted border.
   - Clicking toggles selection (add/remove from `selectedUrls` set).
4. Footer has "Cancel" and "Save Selection" buttons. Save calls `apiPatch("/associations/${associationId}", { selected_image_urls: Array.from(selectedUrls) })`. On success, call `onImagesUpdated(response)` and close. On error, show toast.
5. Show `Loader2` spinner while loading images. Show "No images available" if the API returns an empty array.
6. Header: "Select Images — {productTitle}" or "Select Images" if title is null.

**UI layout:**
- Use Shadcn Dialog (DialogContent with `className="max-w-2xl max-h-[80vh]"`).
- DialogHeader with title.
- ScrollArea wrapping the image grid.
- DialogFooter with Cancel and Save buttons.

**Imports to use:**
- Same Dialog/Button/ScrollArea imports as Task 1
- `apiGet, apiPatch, handleApiError` from `@/lib/api`
- `toast` from `sonner`
- `Check, Loader2, Images` from `lucide-react`

Note: `apiPatch` may not exist yet in `api.ts`. If so, add it following the same pattern as `apiPost` — it's a simple wrapper around `apiFetch` with method "PATCH". Check `frontend/src/lib/api.ts` first; if it already exports `apiPatch`, use it. If not, add it:

```typescript
export async function apiPatch(endpoint: string, data: unknown, options?: FetchOptions) {
  const res = await apiFetch(endpoint, {
    method: "PATCH",
    body: JSON.stringify(data),
    ...options,
  });
  return res.json();
}
```
