```python
@pytest.fixture
def client(mock_settings):
    """FastAPI TestClient with mocked Supabase and auth disabled."""
    from unittest.mock import patch, MagicMock, AsyncMock

    # Mock get_supabase to return None (forces in-memory fallback)
    # Mock get_profile_context to return dev profile
    # Mock slowapi limiter to not rate-limit in tests
    with patch("app.db.get_supabase", return_value=None), \
         patch("app.rate_limit.limiter._check_request_limit", new_callable=AsyncMock):
        from app.main import app
        from fastapi.testclient import TestClient
        # Ensure auth is disabled for tests
        with patch("app.api.auth.get_settings", return_value=mock_settings):
            yield TestClient(app)
```

IMPORTANT: The TestClient must be created INSIDE the patches so the app module picks up mocked services. May need to adjust import order — if `app.main` eagerly initializes Supabase, patch `app.db.get_supabase` BEFORE importing app.main (use `importlib.reload` if needed, or restructure the fixture to patch at module level).

Alternative approach if eager init causes issues: patch at a lower level:
