<objective>
Enforce minimum subtitle duration and ensure video timeline covers the full audio track.

Purpose: Two related bugs cause subtitle cutoff and invisible entries:
1. The silence remover can produce character timestamps where start==end for very short phonemes, creating zero-duration SRT entries that flash invisibly on screen.
2. The assembled video timeline may end slightly shorter than the audio due to floating-point accumulation in segment durations, causing the last subtitle(s) to be cut off when FFmpeg uses audio duration as the master clock.

Output: Modified tts_subtitle_generator.py with minimum duration enforcement, and modified assembly_service.py with a safety margin to ensure video covers audio.

  ```python
  # Minimum duration for a subtitle entry to be perceptible on screen (in seconds).
  # Entries shorter than this get their end time extended to meet the floor.
  MIN_SUBTITLE_DURATION = 0.1  # 100ms
  ```

  **Modify Step 3 (lines 266-278)** to enforce the minimum:
  1. Before creating each SRT entry, check if `phrase["end"] - phrase["start"] < MIN_SUBTITLE_DURATION`
  2. If so, extend the end time: `phrase["end"] = phrase["start"] + MIN_SUBTITLE_DURATION`
  3. But clamp the extended end to not exceed the next phrase's start time (to avoid overlap). If there's a next phrase, use `min(phrase["start"] + MIN_SUBTITLE_DURATION, next_phrase_start)`.
  4. If after clamping the duration is still zero (start == next_phrase_start), skip the entry entirely and log a warning.
  5. Also skip entries where `phrase["start"] == phrase["end"]` after clamping (truly zero-duration from identical timestamps).

  Specifically, replace the Step 3 loop with:
  ```python
  # Step 3: Phrases to SRT (with minimum duration enforcement)
  MIN_DURATION = 0.1  # 100ms minimum for perceptible display
  srt_entries = []
  srt_index = 1
  for i, phrase in enumerate(phrases):
      start = phrase["start"]
      end = phrase["end"]
      text = phrase["text"]

      # Enforce minimum duration
      if end - start < MIN_DURATION:
          # Extend end, but don't overlap next phrase
          if i + 1 < len(phrases):
              end = min(start + MIN_DURATION, phrases[i + 1]["start"])
          else:
              end = start + MIN_DURATION

      # Skip truly zero-duration entries (can happen when phrases are back-to-back)
      if end - start < 0.001:
          logger.warning(f"Skipping zero-duration subtitle entry: '{text}' at {start:.3f}s")
          continue

      start_time_str = _seconds_to_srt_time(start)
      end_time_str = _seconds_to_srt_time(end)

      srt_entry = f"{srt_index}\n{start_time_str} --> {end_time_str}\n{text}\n"
      srt_entries.append(srt_entry)
      srt_index += 1
  ```
