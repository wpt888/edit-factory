<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` with two jobs that run in parallel.

Job 1 -- backend:
- Name: "Python (lint + type-check + test)"
- runs-on: ubuntu-latest
- Steps: checkout v4, setup-python v5 (3.11, cache pip), install system deps (libmagic1 ffmpeg), pip install -r requirements.txt, pip install ruff mypy, ruff check, mypy check, pytest
- Ruff config: `ruff check app/ --select=E,F,W --ignore=E501,E402,W291,W292,W293` (lenient -- codebase has long lines and import ordering)
- Mypy config: `mypy app/ --ignore-missing-imports --no-strict-optional --allow-untyped-defs` (permissive -- codebase is not fully typed)
- Pytest: `pytest tests/ -x -q --tb=short` with env AUTH_DISABLED=true

Job 2 -- frontend:
- Name: "Next.js (lint + type-check)"
- runs-on: ubuntu-latest
- defaults.run.working-directory: frontend
- Steps: checkout v4, setup-node v4 (node 20, cache npm with cache-dependency-path frontend/package-lock.json), npm ci, npm run lint, npm run typecheck

Design decisions:
- Two parallel jobs for faster feedback
- Python 3.11 (stable, widely used)
- ruff for Python linting (fast, replaces flake8/pycodestyle)
- mypy with permissive flags since codebase is not fully typed
- AUTH_DISABLED=true for pytest so tests skip Supabase JWT validation
- libmagic1 and ffmpeg installed as system deps for tests
- Playwright E2E tests NOT included (require dev server + Supabase)
- ruff and mypy installed separately (dev-only tools, not in requirements.txt)

Branch protection note: After this workflow exists and passes, user should enable branch protection rules on GitHub (Settings then Branches then main then Require status checks). This is a manual GitHub UI step.
