**Types:**
```typescript
type QueueItemStatus = "waiting" | "processing" | "done" | "failed";

interface QueueItem {
  id: string;           // unique ID (crypto.randomUUID or counter)
  file: File;
  status: QueueItemStatus;
  progress: number;     // 0-100
  error?: string;
  projectId?: string;   // created project ID
  clipCount?: number;   // number of clips generated
}
```

**Component: `BatchUploadQueue`**

Props:
```typescript
interface BatchUploadQueueProps {
  variantCount?: number;  // default 3
}
```

**UI Layout:**
1. **Drop zone** at the top — accepts multiple video files via drag-and-drop or file picker (input with `multiple` attribute). Accepts video/mp4, video/webm, video/quicktime.

2. **Queue list** below the drop zone — each item shows:
   - Video filename + file size (formatted: "2.4 MB")
   - Status badge: waiting (gray), processing (blue with spinner), done (green with checkmark), failed (red with X)
   - Progress bar (visible during "processing" status)
   - Error message (visible when "failed")
   - "Remove" button (visible when "waiting" — removes from queue)
   - "Retry" button (visible when "failed" — resets to "waiting")
   - Clip count (visible when "done")

3. **Controls** above the queue list:
   - "Start Processing" button — begins sequential processing of all "waiting" items
   - "Clear Completed" button — removes "done" items from the list
   - "Cancel" button — stops processing after current item finishes (does not abort in-flight)
   - Queue summary: "3 waiting, 1 processing, 2 done, 0 failed"

**Processing logic:**
- When "Start Processing" is clicked, iterate through queue items with status "waiting" one at a time
- For each item:
  1. Set status to "processing"
  2. Create a project: POST /api/v1/library/projects with body `{name: "Batch - {filename}", description: "Batch uploaded"}`
  3. Upload + generate: POST /api/v1/library/projects/{projectId}/generate-raw as FormData with the video file and variant_count
  4. Poll job status via GET /api/v1/jobs/{jobId} every 3 seconds until complete or failed
  5. On success: set status to "done", store clipCount
  6. On failure: set status to "failed", store error message
  7. Move to next "waiting" item
- Use a ref (`processingRef`) to track whether processing is active, so "Cancel" can set it to false and the loop stops after the current item

**State management:**
- `queue: QueueItem[]` — the list of items
- `isProcessing: boolean` — whether the sequential processor is running
- Use `useRef` for the processing loop control to avoid stale closures

**Drag-and-drop:**
- Use native HTML5 drag events (onDragOver, onDragLeave, onDrop)
- Show visual feedback (border color change) when dragging over
- Prevent duplicates: skip files with same name+size already in queue

**Styling:** Use Shadcn Card for each queue item, Badge for status, Progress component for progress bar. Use Tailwind for the drop zone styling (dashed border, hover states).

Export as named export: `export function BatchUploadQueue`.
