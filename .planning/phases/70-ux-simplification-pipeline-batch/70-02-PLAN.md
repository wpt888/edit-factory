  <action>
Create `frontend/src/components/simple-mode-pipeline.tsx` — a self-contained 3-step pipeline component.

**Props interface:**
```typescript
interface SimplePipelineProps {
  onSwitchToAdvanced: () => void;  // callback to toggle mode
}
```

**3-step flow:**

**Step 1 — Upload Video:**
- Drag-and-drop zone (dashed border, accepts video/mp4, video/webm, video/mov)
- Shows file name + size after selection
- "Remove" button to clear selection
- "Next" button enabled when file is selected
- Text: "Drop your video here or click to browse"

**Step 2 — Choose Style:**
- Grid of 3-5 preset cards (from STYLE_PRESETS imported from types/pipeline-presets)
- Each card shows: icon (use lucide-react icon by name), preset name, description
- Selected card has primary border/highlight
- "Back" and "Generate" buttons
- Clicking "Generate":
  1. Creates a project via POST /api/v1/library/projects (name: "Simple - {preset.name} - {timestamp}")
  2. Uploads the video to POST /api/v1/library/projects/{project_id}/generate-raw with the file and variant_count from preset
  3. Then generates a script via POST /api/v1/pipeline/generate with idea auto-derived from preset name (e.g., "Create an energetic short video" for energetic_short preset)
  4. Auto-applies preset params as pipeline settings (voice speed, stability, etc.)
  5. Shows a loading spinner during the entire process
  6. On success, transitions to Step 3

**Step 3 — Download:**
- Shows processing status by polling the pipeline/job status
- When complete, shows download buttons for each rendered variant
- "Download All" button
- "Start Over" button to reset to Step 1
- Download uses existing library clip download endpoint

**State management:**
- `simpleStep` (1 | 2 | 3), `selectedPreset`, `uploadedFile`, `isProcessing`, `error`, `projectId`, `pipelineId`, `completedVariants`
- Persist mode preference to localStorage key `ef_pipeline_mode`

**Styling:** Use existing Shadcn/UI components (Card, Button, Badge). Use Tailwind grid for preset cards. Keep the component visually clean — no technical jargon.

**Important:** The component manages its own API calls. It does NOT reuse the pipeline page's state (that would couple simple and advanced modes). It imports only from `@/lib/api`, `@/types/pipeline-presets`, and Shadcn components.

For the actual background processing and polling, reuse the pattern from the pipeline page: poll job status via GET /api/v1/jobs/{job_id} or GET /api/v1/pipeline/status/{pipeline_id}.
