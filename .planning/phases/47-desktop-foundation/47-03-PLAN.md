- `app/main.py` line 100-101: lifespan logs `AUTH_DISABLED` warning -- should also warn for desktop mode
- Research file: see Pattern 3 (Desktop Mode in Auth) and Pattern 4 (FFmpeg Bundled Binary)

<task id="C2">
<title>Refactor FFmpeg path setup in main.py to support bundled binary</title>
<details>
In `app/main.py`, replace lines 8-11 (the FFmpeg PATH injection block at the top of the file):

**Current code:**
```python
# Add local FFmpeg to PATH if exists
_ffmpeg_bin = Path(__file__).parent.parent / "ffmpeg" / "ffmpeg-master-latest-win64-gpl" / "bin"
if _ffmpeg_bin.exists():
    os.environ['PATH'] = str(_ffmpeg_bin) + os.pathsep + os.environ.get('PATH', '')
```

**Replace with:**
```python
# Add FFmpeg to PATH: check bundled location (desktop mode) then local dev location
def _setup_ffmpeg_path():
    desktop_mode = os.getenv("DESKTOP_MODE", "").lower() in ("true", "1", "yes")
    candidates = []
    if desktop_mode:
        # Desktop mode: bundled FFmpeg in AppData (placed by installer, Phase 52)
        appdata = os.getenv("APPDATA")
        if appdata:
            candidates.append(Path(appdata) / "EditFactory" / "bundled" / "ffmpeg" / "bin")
    # Dev fallback: local win64-gpl checkout in project root
    candidates.append(Path(__file__).parent.parent / "ffmpeg" / "ffmpeg-master-latest-win64-gpl" / "bin")
    for candidate in candidates:
        if candidate.exists():
            os.environ['PATH'] = str(candidate) + os.pathsep + os.environ.get('PATH', '')
            break

_setup_ffmpeg_path()
```

**IMPORTANT:** This code runs at module import time (before `app.config` may be imported by other modules), so it reads `DESKTOP_MODE` and `APPDATA` directly from `os.getenv()` rather than from Settings. This is intentional -- FFmpeg must be on PATH before any service that shells out to `ffmpeg` is imported.
</details>
</task>
