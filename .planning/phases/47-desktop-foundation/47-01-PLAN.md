## Goal

Replace the hardcoded `_BASE_DIR = Path(__file__).parent.parent` in `app/config.py` with an `APP_BASE_DIR` abstraction that resolves to `%APPDATA%\EditFactory\` when `DESKTOP_MODE=true` and falls back to the project root in dev mode. Add `desktop_mode` field to Settings. Update `ensure_dirs()` to create the AppData directory structure on first run. Override `settings_customise_sources` so Settings loads from AppData `.env` first, project `.env` second.

```python
def _get_app_base_dir() -> Path:
    """Returns %APPDATA%\\EditFactory in desktop mode, project root in dev."""
    if os.getenv("DESKTOP_MODE", "").lower() in ("true", "1", "yes"):
        appdata = os.getenv("APPDATA")
        if appdata:
            base = Path(appdata) / "EditFactory"
            base.mkdir(parents=True, exist_ok=True)
            return base
    # Dev / WSL / CI: use project root (existing behaviour)
    return Path(__file__).parent.parent

_BASE_DIR = _get_app_base_dir()
```

<details>
In `app/config.py`, replace `class Config: env_file = ".env"` with:

```python

    env_file=None,  # Disable default; controlled in settings_customise_sources

)
```

Add the import at the top: `from pydantic_settings import BaseSettings, SettingsConfigDict`

Add this classmethod to `Settings`:

```python
@classmethod
def settings_customise_sources(
    cls,
    settings_cls,
    init_settings,
    env_settings,
    dotenv_settings,
    file_secret_settings,
):
    """Load env vars, then AppData .env (desktop), then project .env (dev fallback)."""
    sources = [init_settings, env_settings]
    # Desktop mode: AppData .env has priority
    appdata_env = _BASE_DIR / ".env"
    desktop = os.getenv("DESKTOP_MODE", "").lower() in ("true", "1", "yes")
    if desktop and appdata_env.exists():
        from pydantic_settings import DotEnvSettingsSource
        sources.append(DotEnvSettingsSource(settings_cls, env_file=appdata_env))
    # Always include project .env as lowest-priority fallback
    project_env = Path(__file__).parent.parent / ".env"
    if project_env.exists():
        from pydantic_settings import DotEnvSettingsSource
        sources.append(DotEnvSettingsSource(settings_cls, env_file=project_env))
    return tuple(sources)
```

**IMPORTANT:** The `DotEnvSettingsSource` import path may vary by pydantic-settings version. If `from pydantic_settings import DotEnvSettingsSource` fails, try `from pydantic_settings.env_settings import DotEnvSettingsSource` or `from pydantic_settings.main import DotEnvSettingsSource`. Use a try/except import block if needed.

**IMPORTANT:** The `settings_customise_sources` signature varies between pydantic-settings versions. In older versions the parameter is `file_secret_settings`, in newer it may be `secrets_settings`. Match the signature from the installed version. Check with: `python -c "import pydantic_settings; print(pydantic_settings.__version__)"`
</details>
