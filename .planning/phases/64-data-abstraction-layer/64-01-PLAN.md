<task type="auto">
  <name>Task 2: Create SQLite schema from Supabase migrations</name>
  <files>supabase/sqlite_schema.sql</files>
  <action>
Read ALL 26 Supabase migration files in `supabase/migrations/` and produce a single `supabase/sqlite_schema.sql` file with equivalent SQLite CREATE TABLE statements.

**Type mapping rules:**
- `UUID` -> `TEXT` (store as string)
- `TIMESTAMPTZ` / `TIMESTAMP WITH TIME ZONE` -> `TEXT` (ISO8601 format)
- `JSONB` / `JSON` -> `TEXT` (store as JSON string)
- `TEXT[]` (array) -> `TEXT` (store as JSON array string)
- `SERIAL` / `BIGSERIAL` -> `INTEGER PRIMARY KEY AUTOINCREMENT`
- `BOOLEAN` -> `INTEGER` (0/1)
- `DOUBLE PRECISION` / `FLOAT` -> `REAL`
- `DATE` -> `TEXT` (ISO8601 date)
- `TIME` -> `TEXT` (HH:MM:SS)
- `INTEGER` -> `INTEGER`
- `TEXT` -> `TEXT`

**Tables to include (from codebase grep):**
editai_projects, editai_clips, editai_clip_content, editai_project_segments, editai_segments, editai_source_videos, editai_pipelines, editai_assembly_jobs, editai_export_presets, editai_exports, editai_postiz_publications, editai_product_groups, editai_generation_progress, editai_tts_assets, editai_schedule_plans, editai_schedule_items, profiles, jobs, api_costs, product_feeds, products, elevenlabs_accounts, generated_images, image_prompt_templates, segment_product_associations

**What to SKIP:**
- All RLS policies (SQLite has no RLS)
- All `auth.users` foreign key references (replace with plain TEXT user_id column, no FK)
- All `REFERENCES auth.users(id)` clauses — remove them
- PostgreSQL-specific syntax: `gen_random_uuid()`, `now()`, `DO $$ ... $$` blocks
- Views (`v_catalog_products`, `v_catalog_products_grouped`) — create them as separate CREATE VIEW statements if straightforward, or skip
- Indexes: include only primary keys and the most important unique/foreign key indexes

**What to ADD:**
- Header comment explaining this file creates all tables for SQLite backend
- `DEFAULT (lower(hex(randomblob(4)) || ...))` for UUID defaults, OR just leave no default and let Python generate UUIDs
- `DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` for timestamp defaults
- Each CREATE TABLE should be `IF NOT EXISTS`
- Add a `schema_version` table: `CREATE TABLE IF NOT EXISTS schema_version (version INTEGER PRIMARY KEY, applied_at TEXT DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')));`
- Insert row: `INSERT OR IGNORE INTO schema_version (version) VALUES (1);`

Read each migration file sequentially to build up the full column set per table (some migrations ALTER existing tables to add columns). The final CREATE TABLE should include ALL columns from ALL migrations for that table.

For the `profiles` table, consolidate columns from migrations 002, 006, 014, and 020.
For `editai_clips`, consolidate from the original schema plus migrations 009, 012, 015, 024, 025.
For `editai_pipelines`, consolidate from 016 plus 021.

The output file must be a valid SQLite script that can run with: `sqlite3 test.db < supabase/sqlite_schema.sql`
