Create a SQL migration that:

1. **Enables RLS** on all 13 editai_* tables (reverse of migration 022).

2. **Creates profile-based policies** for the `authenticated` role on each table. Use the pattern from migration 005 as a template:
   - Tables with `profile_id` column directly (editai_projects, editai_pipelines, editai_segments, editai_source_videos, editai_export_presets): Policy checks `profile_id IN (SELECT id FROM profiles WHERE user_id = (SELECT auth.uid()))`.
   - Tables that inherit through `project_id` (editai_clips, editai_clip_content, editai_project_segments, editai_postiz_publications, editai_product_groups): Policy joins through `editai_projects` to check profile ownership.
   - Tables that inherit through `clip_id` (editai_clip_content): Policy joins through `editai_clips -> editai_projects`.
   - Tables with `pipeline_id` (editai_assembly_jobs): Policy joins through `editai_pipelines`.
   - editai_tts_assets: Check if it has `profile_id` column; if so, use direct profile check.
   - editai_exports: Check structure; likely joins through project or clip.

3. **Creates service_role bypass policies** for each table:
   ```sql
   CREATE POLICY "Service role full access" ON table_name
     FOR ALL
     TO service_role
     USING (true)
     WITH CHECK (true);
   ```
   Note: The `service_role` role in Supabase inherits the `authenticated` role, but service_role policies with `TO service_role` take precedence.

4. **Drop existing policies first** (idempotent) — drop any leftover policies from migrations 001 and 005 before creating new ones.

5. **Verify RLS** at the end with a DO block that checks `pg_tables.rowsecurity = true` for all 13 tables.

IMPORTANT: Before writing policies for tables other than editai_projects, editai_clips, editai_clip_content, and editai_project_segments (which are covered in migration 005), inspect the table structure by running queries against Supabase to determine the correct join path. Use `mcp__supabase__execute_sql` with `SELECT column_name FROM information_schema.columns WHERE table_name = 'editai_tts_assets'` to discover column structure.

Key considerations:
- Use `(SELECT auth.uid())` wrapper (not bare `auth.uid()`) to avoid re-evaluation per row.
- Wrap all policy creation in a transaction.
- The `service_role` bypass is critical — the backend uses this key and must not be blocked by RLS.
