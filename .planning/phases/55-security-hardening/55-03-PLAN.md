Add a new function `sanitize_srt_for_ffmpeg` to `app/services/srt_validator.py` that sanitizes SRT **content** (not path) for safe FFmpeg subtitle rendering.

**Understanding the problem:** FFmpeg's `subtitles` filter parses SRT files using libass. The `force_style` parameter and filter syntax use characters like `'`, `\`, `:`, `{`, `}`, `[`, `]` as control characters. When these appear in the SRT TEXT content, they can break the filter parsing.

However, there's an important subtlety: The SRT content is read by libass as a separate file — the special character issue is primarily in:
1. **ASS override tags**: `{\b1}` — curly braces in SRT text could be interpreted as ASS override tags by libass
2. **Backslash sequences**: `\N` (newline), `\n` (soft newline), `\h` (hard space) are ASS control sequences

The FFmpeg filter syntax escaping (colons, brackets, apostrophes) applies to the `-vf` command string, NOT to the content of the SRT file itself. So the SRT file content escaping is specifically about:
- **Curly braces `{}`**: Escape as `\{` and `\}` to prevent ASS override tag interpretation
- **Backslashes `\`**: Escape as `\\` to prevent ASS control sequence interpretation (e.g., `\N` being treated as newline)

For apostrophes and colons IN THE SRT FILE CONTENT: These are safe because they're inside the SRT file read by libass, not in the FFmpeg filter string. The path escaping in video_processor.py handles the filter string escaping.

**Implementation:**

```python
def sanitize_srt_for_ffmpeg(srt_content: str) -> str:
    """Escape special characters in SRT text content for safe FFmpeg/libass rendering.

    Prevents ASS override tag injection and control sequence interpretation
    in subtitle text. Only processes text lines — SRT structure (timestamps,
    sequence numbers, blank lines) is preserved.

    Characters escaped:
    - Backslashes: \\ -> \\\\ (prevents \\N, \\n, \\h interpretation)
    - Curly braces: { -> \\{, } -> \\} (prevents ASS override tags like {\\b1})

    Args:
        srt_content: Raw SRT file content

    Returns:
        SRT content with text lines escaped for FFmpeg/libass
    """
    if not srt_content:
        return srt_content

    lines = srt_content.split('\n')
    result = []
    i = 0
    while i < len(lines):
        line = lines[i]
        stripped = line.strip()

        # Sequence number (digit-only line)
        if stripped.isdigit():
            result.append(line)
            i += 1
            continue

        # Timestamp line (contains " --> ")
        if ' --> ' in stripped:
            result.append(line)
            i += 1
            continue

        # Empty line (SRT entry separator)
        if not stripped:
            result.append(line)
            i += 1
            continue

        # Text line — sanitize it
        # Order matters: escape backslashes FIRST, then braces
        sanitized = line.replace('\\', '\\\\')
        sanitized = sanitized.replace('{', '\\{')
        sanitized = sanitized.replace('}', '\\}')
        result.append(sanitized)
        i += 1

    return '\n'.join(result)
```
