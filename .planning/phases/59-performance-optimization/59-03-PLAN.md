# Plan 59-03: SSE Job Progress (Replace Polling)

<objective>
After this plan, job progress updates arrive via Server-Sent Events instead of HTTP polling. The browser opens a single persistent SSE connection per job, and the server pushes progress events as they occur. This eliminates the 2-second polling delay and reduces HTTP overhead.

Purpose: Real-time progress without polling overhead.
Output: SSE endpoint for job streaming + rewritten frontend hooks using EventSource.
</objective>

**Current state:**
- `useJobPolling` hook in use-job-polling.ts polls `GET /jobs/{jobId}` every 2 seconds via setTimeout
- `useBatchPolling` hook polls `GET /products/batch/{batchId}/status` every 2 seconds
- `usePolling` is a generic polling primitive used by assembly status
- Frontend consumers: library page (render dialog), product-video page, batch-generate page, progress-tracker component
- Backend GET /jobs/{job_id} at routes.py line ~549 returns current job state from JobStorage
- JobStorage uses Supabase with in-memory fallback — updates happen in background tasks

**SSE design notes:**
- FastAPI supports SSE via `StreamingResponse` with `media_type="text/event-stream"`
- The SSE endpoint will internally poll JobStorage every 1 second (server-side loop) and yield events when progress changes
- SSE format: `event: <type>\ndata: <json>\n\n`
- Event types: `progress`, `completed`, `failed`, `heartbeat`
- Connection stays open until job reaches terminal state (completed/failed) or client disconnects

**Backend: SSE streaming endpoint for job progress**

In `app/api/routes.py`:

1. Add imports at top:
   ```python
   import asyncio
   from fastapi.responses import StreamingResponse
   import json as json_lib
   ```

2. Add the SSE endpoint BEFORE the existing `get_job` endpoint (to avoid route conflicts with `{job_id}`):

   ```python
   @router.get("/jobs/{job_id}/stream")
   async def stream_job_progress(job_id: str, request: Request):
       """Stream job progress via Server-Sent Events."""
       job = get_job_storage().get_job(job_id)
       if not job:
           raise HTTPException(status_code=404, detail="Job not found")

       async def event_generator():
           last_progress = None
           last_status = None
           heartbeat_counter = 0

           while True:
               # Check if client disconnected
               if await request.is_disconnected():
                   break

               current_job = get_job_storage().get_job(job_id)
               if not current_job:
                   yield f"event: failed\ndata: {json_lib.dumps({'job_id': job_id, 'error': 'Job not found'})}\n\n"
                   break

               current_progress = current_job.get("progress")
               current_status = current_job.get("status")

               # Only send event when something changed
               if current_progress != last_progress or current_status != last_status:
                   last_progress = current_progress
                   last_status = current_status

                   if current_status == "completed":
                       # Build result payload
                       result_data = current_job.get("result")
                       # For assembly jobs, include final_video_path
                       if current_job.get("job_type") == "assembly" and current_job.get("final_video_path"):
                           result_data = {"final_video_path": current_job["final_video_path"]}
                       payload = {
                           "job_id": job_id,
                           "status": "completed",
                           "progress": "100",
                           "result": result_data,
                       }
                       yield f"event: completed\ndata: {json_lib.dumps(payload)}\n\n"
                       break

                   elif current_status == "failed":
                       payload = {
                           "job_id": job_id,
                           "status": "failed",
                           "error": current_job.get("error", "Unknown error"),
                       }
                       yield f"event: failed\ndata: {json_lib.dumps(payload)}\n\n"
                       break

                   else:
                       payload = {
                           "job_id": job_id,
                           "status": current_status,
                           "progress": current_progress or "",
                       }
                       yield f"event: progress\ndata: {json_lib.dumps(payload)}\n\n"

               # Heartbeat every 15 seconds to keep connection alive
               heartbeat_counter += 1
               if heartbeat_counter >= 15:
                   heartbeat_counter = 0
                   yield f"event: heartbeat\ndata: {json_lib.dumps({'ts': int(asyncio.get_event_loop().time())})}\n\n"

               await asyncio.sleep(1)

       return StreamingResponse(
           event_generator(),
           media_type="text/event-stream",
           headers={
               "Cache-Control": "no-cache",
               "Connection": "keep-alive",
               "X-Accel-Buffering": "no",  # Disable nginx buffering if behind proxy
           },
       )
   ```

3. **Route placement:** Put this endpoint AFTER the `/jobs` list endpoint (line ~597) but BEFORE or alongside the existing `/jobs/{job_id}` GET endpoint. Since this has a more specific path (`/jobs/{job_id}/stream`), FastAPI will match it correctly regardless of order — but place it near the other job endpoints for clarity.

4. **No auth on SSE endpoint**: The SSE endpoint does NOT use `Depends(get_profile_context)` because EventSource in browsers cannot send custom headers. This is acceptable because:
   - Job IDs are UUIDs (unguessable)
   - The endpoint is read-only
   - The regular GET endpoint with auth still exists for programmatic access
   - If auth is needed later, use query parameter tokens

**Verification:** `curl -N http://localhost:8000/api/v1/jobs/<job_id>/stream` should show SSE events streaming.

**Frontend: Rewrite useJobPolling to use EventSource**

In `frontend/src/hooks/use-job-polling.ts`:

2. Replace the internal `setTimeout`-based polling with `EventSource`:

   ```typescript
   const startPolling = useCallback((jobId: string) => {
     // Reset state
     isCancelledRef.current = false;
     setCurrentJob(null);
     setIsPolling(true);
     setProgress(10);
     setStatusText("pending");
     setElapsedTime(0);
     setEstimatedRemaining("Calculez...");
     startTimeRef.current = Date.now();

     // Start elapsed time counter (keep as-is)
     elapsedIntervalRef.current = setInterval(() => {
       if (startTimeRef.current) {
         setElapsedTime(Math.floor((Date.now() - startTimeRef.current) / 1000));
       }
     }, 1000);

     // Open SSE connection
     const apiBase = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000/api/v1";
     const url = `${apiBase}/jobs/${jobId}/stream`;
     const eventSource = new EventSource(url);
     eventSourceRef.current = eventSource;

     eventSource.addEventListener("progress", (e) => {
       const data = JSON.parse(e.data);
       const job: Job = { job_id: data.job_id, status: data.status, progress: data.progress, error: null, result: null, created_at: "", updated_at: "" };
       setCurrentJob(job);
       const progressNum = extractProgress(job);
       setProgress(progressNum);
       setStatusText(data.status);
       const elapsed = startTimeRef.current ? Math.floor((Date.now() - startTimeRef.current) / 1000) : 0;
       setEstimatedRemaining(calculateETA(progressNum, elapsed));
       onProgressRef.current?.(progressNum, data.status, job);
     });

     eventSource.addEventListener("completed", (e) => {
       const data = JSON.parse(e.data);
       setProgress(100);
       setStatusText("completed");
       onCompleteRef.current?.(data.result);
       cleanup();
     });

     eventSource.addEventListener("failed", (e) => {
       const data = JSON.parse(e.data);
       onErrorRef.current?.(data.error || "Job failed");
       cleanup();
     });

     eventSource.onerror = () => {
       // EventSource auto-reconnects by default
       // If we explicitly closed, don't log
       if (!isCancelledRef.current) {
         console.warn("SSE connection error, will auto-reconnect");
       }
     };
   }, [calculateETA]);
   ```

   ```typescript
     }
     if (elapsedIntervalRef.current) {
       clearInterval(elapsedIntervalRef.current);
       elapsedIntervalRef.current = null;
     }
   }, []);
   ```

   Rename old `stopPolling` to `cleanup` internally, keep `stopPolling` as the exported name that calls `cleanup`.

5. Keep `extractProgress` and `formatElapsedTime` exports unchanged.
