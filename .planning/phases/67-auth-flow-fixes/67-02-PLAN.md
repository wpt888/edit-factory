  <name>Task 2: Add forgot password flow to login page</name>
  <files>frontend/src/app/login/page.tsx, frontend/src/app/auth/callback/route.ts</files>

1. **login/page.tsx** — Add a "Forgot password?" section:
   - Add a state variable `forgotMode` (boolean, default false) and `resetSent` (boolean, default false)
   - Below the password field (or as an alternative view), add a "Forgot password?" link that sets `forgotMode = true`
   - When `forgotMode` is true, show only the email field and a "Send Reset Link" button. Hide the password field and Sign In button.
   - On submit in forgot mode, call `supabase.auth.resetPasswordForEmail(email, { redirectTo: window.location.origin + '/auth/callback?next=/login/reset-password' })`
   - On success, set `resetSent = true` and show a success message: "Check your email for a password reset link"
   - Add a "Back to sign in" link to return to normal login mode
   - Handle errors (invalid email, rate limiting) with the existing Alert component

2. **Create `/login/reset-password/page.tsx`** — Password reset form:
   - Create file at `frontend/src/app/login/reset-password/page.tsx`
   - This page loads after user clicks the reset link in their email, which redirects through `/auth/callback` and lands here with an active session
   - Show two fields: "New password" and "Confirm password"
   - On submit, call `supabase.auth.updateUser({ password: newPassword })`
   - Validate passwords match and minimum length (6 chars)
   - On success, show "Password updated" message and redirect to /login after 2 seconds
   - Use the same Card styling as the login page for visual consistency
   - Import same components: Card, CardHeader, CardContent, CardFooter, Button, Input, Label, Alert

3. **auth/callback/route.ts** — The existing callback already handles code exchange. Supabase password reset uses the same PKCE flow, so the existing code should work. The `next` query param will be `/login/reset-password`, directing the user to the password reset form after code exchange. No changes needed unless the recovery flow uses a different mechanism. Verify by reading Supabase docs pattern: resetPasswordForEmail sends email with link to redirect URL containing a code, which gets exchanged via the existing callback route.
