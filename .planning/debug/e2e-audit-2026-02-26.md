## Resolution

root_cause: FastAPI route ordering bug in app/api/segments_routes.py — GET /{segment_id} was registered at line 915 (before GET /product-groups-bulk at line 1523), causing FastAPI to route all requests for /segments/product-groups-bulk to get_segment() with segment_id="product-groups-bulk", which then failed when Supabase tried to use it as a UUID.

fix: Moved the @router.get("/product-groups-bulk") endpoint to line 912 (before @router.get("/{segment_id}") at line 971). Added try/except error handling. Removed the original duplicate at the bottom. Added docstring explaining the ordering constraint.

verification: After fix, curl to http://127.0.0.1:8002/api/v1/segments/product-groups-bulk?source_video_ids=5725c6f1-610b-4793-9545-42f695a0c477 returned HTTP 200 (was HTTP 500).
