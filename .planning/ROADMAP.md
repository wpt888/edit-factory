### Phase 80: Library routes repository migration

**Goal**: Every `repo.get_client()` call in `app/api/library_routes.py` (27 sites covering `/library/clips/{id}/srt`, `/audio`, `/download`, `/render`, `/regenerate-voiceover`, `/remove-audio`, `/restore`, `/permanent`, `/content`, `/tags`, `/all-clips`, `/trash`, `/projects/{id}/generate`, `/projects/{id}/generate-from-segments`, `/clips/bulk-delete`, `/clips/bulk-render`, `/sync-orphans`) is replaced with typed repository methods or `table_query(QueryFilters)` calls. Routes that previously returned `503 Database not available` under `DATA_BACKEND=sqlite` now return `200` (or the correct status for the operation).

**Depends on**: Nothing (first v13 phase).
**Requirements**: FUNC-01, FUNC-03.
**Success Criteria**:
  1. Zero `repo.get_client()` calls remain in `library_routes.py` ‚Äî `grep -c "get_client()" app/api/library_routes.py` returns `0`.
  2. A `ROUTES-AUDIT.md` artifact lists each migrated call site with the pattern (A/B/C/D from ARCHITECTURE.md ¬ß1) and the repository method used.
  3. New ABC methods added to `app/repositories/base.py` are implemented in both `SupabaseRepository` and `SQLiteRepository` ‚Äî no `NotImplementedError` paths.
  4. Existing routes still work in Supabase mode ‚Äî regression test suite passes.
  5. Each migrated route gains a pytest case asserting `200` under `DATA_BACKEND=sqlite`.

**Plans**: 3 plans (Wave 1 ‚Üí 2 ‚Üí 3, sequential because all three plans modify app/api/library_routes.py):
  - 80-01-PLAN.md ‚Äî Audit + new ABC methods + Pattern A/B migration
  - 80-02-PLAN.md ‚Äî Pattern C/D migration + helper refactor + dead-code removal
  - 80-03-PLAN.md ‚Äî Per-route pytest cases asserting 200 under DATA_BACKEND=sqlite
